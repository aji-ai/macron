GitHub Copilot CLI Approach
Prerequisite: You’ll need have installed brew 

1/ Install GitHub Copilot CLI copilot

> brew install copilot-cli@prerelease
2/ Put this file somewhere and call it `copilot-modes-installer.sh`

#!/usr/bin/env bash
set -euo pipefail

# -----------------------------
# Copilot CLI Modes Installer (new `copilot` binary)
# -----------------------------
# Creates: ~/.config/copilot-modes/copilot-modes.zsh
#
# Provides TWO layers:
#
# 1) THIN wrappers (c-*)
#    - Forward Copilot flags exactly (you provide -p/--prompt when desired)
#    - Add defaults (model + allow/deny policy)
#
#      c-ask   -p "..." [copilot flags...]
#      c-plan  -p "..." [copilot flags...]
#      c-safe  -p "..." [copilot flags...]
#      c-agent -p "..." [copilot flags...]
#
# 2) PROGRAMMATIC convenience (cp-*)
#    - Adds -p automatically so it behaves like a Unix filter
#
#      cp-ask   "..."
#      cp-plan  "..."
#      cp-safe  "..."
#      cp-agent "..."
#
# Utility:
#   c-modes : show Mode → Model mapping configured here
#
# Notes:
# - Model availability is account/policy dependent.
#   To verify what you can use, start `copilot` interactively and type `/models`.

ASK_MODEL="claude-haiku-4.5"
PLAN_MODEL="gpt-4.1"
AGENT_MODEL="gpt-4.1"

INSTALL_DIR="$HOME/.config/copilot-modes"
OUT_FILE="$INSTALL_DIR/copilot-modes.zsh"

cat <<EOF
This will set up Copilot CLI helpers for the NEW \`copilot\` binary.

You will get:

THIN wrappers (flags forwarded; you provide -p):
  c-ask    ... advisory only (deny shell + write)
  c-plan   ... plan-first helper (deny shell + write)
  c-safe   ... allow write, deny shell
  c-agent  ... allow all tools, deny dangerous commands

PROGRAMMATIC convenience (adds -p automatically):
  cp-ask, cp-plan, cp-safe, cp-agent

Utility:
  c-modes   ... show Mode → Model mapping (what this installer configured)

Actions this script will take:
  • Create:  $INSTALL_DIR
  • Write:   $OUT_FILE

It will NOT:
  • edit ~/.zshrc
  • modify ~/.copilot/config.json
  • change trusted folders
  • install copilot

Proceed? Type Y to continue:
EOF

read -r confirm
if [[ "${confirm}" != "Y" ]]; then
  echo "Aborted (no changes made)."
  exit 0
fi

if ! command -v copilot >/dev/null 2>&1; then
  echo "⚠️  Warning: 'copilot' is not on PATH in this shell."
  echo "   Install it first or ensure it's available before using the functions."
  echo
fi

mkdir -p "$INSTALL_DIR"

cat > "$OUT_FILE" <<EOF
# Generated by copilot-modes-install.sh
# Source from ~/.zshrc:
#   source "\$HOME/.config/copilot-modes/copilot-modes.zsh"

export COPILOT_MODE_ASK_MODEL="${ASK_MODEL}"
export COPILOT_MODE_PLAN_MODEL="${PLAN_MODEL}"
export COPILOT_MODE_AGENT_MODEL="${AGENT_MODEL}"

# -----------------------
# Utility
# -----------------------

c-modes() {
  echo "Mode → Model mapping:"
  printf "  %-8s %s\\n" "c-ask"   "\${COPILOT_MODE_ASK_MODEL}"
  printf "  %-8s %s\\n" "c-plan"  "\${COPILOT_MODE_PLAN_MODEL}"
  printf "  %-8s %s\\n" "c-safe"  "\${COPILOT_MODE_AGENT_MODEL}"
  printf "  %-8s %s\\n" "c-agent" "\${COPILOT_MODE_AGENT_MODEL}"
  echo "Convenience aliases:"
  printf "  %-8s %s\\n" "cp-ask"   "c-ask  -p"
  printf "  %-8s %s\\n" "cp-plan"  "c-plan -p"
  printf "  %-8s %s\\n" "cp-safe"  "c-safe -p"
  printf "  %-8s %s\\n" "cp-agent" "c-agent -p"
}

# Raw interactive entry (no defaults)
c() { copilot "\$@"; }

# -----------------------
# THIN WRAPPERS (c-*)
# -----------------------

# ASK: advisory only (deny shell + write)
c-ask() {
  COPILOT_MODEL="\${COPILOT_MODE_ASK_MODEL}" \\
  copilot "\$@" \\
    --deny-tool 'shell' \\
    --deny-tool 'write'
}

# PLAN: deny shell+write, and prefix prompt when -p/--prompt is used.
# This keeps flags forwarding intact while making plan-mode easy.
c-plan() {
  local args=()
  local i=1
  local plan_prefix="Plan first. Ask clarifying questions. Produce a structured plan before any edits."

  while [[ \$i -le \$# ]]; do
    local a="\${!i}"
    if [[ "\$a" == "-p" || "\$a" == "--prompt" ]]; then
      local j=\$((i+1))
      local p="\${!j:-}"
      args+=("\$a" "\$plan_prefix \$p")
      i=\$((i+2))
      continue
    fi
    args+=("\$a")
    i=\$((i+1))
  done

  COPILOT_MODEL="\${COPILOT_MODE_PLAN_MODEL}" \\
  copilot "\${args[@]}" \\
    --deny-tool 'shell' \\
    --deny-tool 'write'
}

# SAFE: allow write, deny shell
c-safe() {
  COPILOT_MODEL="\${COPILOT_MODE_AGENT_MODEL}" \\
  copilot "\$@" \\
    --allow-tool 'write' \\
    --deny-tool 'shell'
}

# AGENT: allow broadly, deny dangerous commands (deny wins)
c-agent() {
  COPILOT_MODEL="\${COPILOT_MODE_AGENT_MODEL}" \\
  COPILOT_ALLOW_ALL="true" \\
  copilot "\$@" \\
    --allow-all-tools \\
    --deny-tool 'shell(rm)' \\
    --deny-tool 'shell(rm:*)' \\
    --deny-tool 'shell(chmod)' \\
    --deny-tool 'shell(chmod:*)' \\
    --deny-tool 'shell(chown)' \\
    --deny-tool 'shell(chown:*)' \\
    --deny-tool 'shell(sudo)' \\
    --deny-tool 'shell(sudo:*)' \\
    --deny-tool 'shell(git push)' \\
    --deny-tool 'shell(git push:*)'
}

# -----------------------
# PROGRAMMATIC CONVENIENCE (cp-*)
# -----------------------
# These are just: c-<mode> -p "<prompt>"

cp-ask()   { c-ask   -p "\$*"; }
cp-plan()  { c-plan  -p "\$*"; }
cp-safe()  { c-safe  -p "\$*"; }
cp-agent() { c-agent -p "\$*"; }
EOF

echo
echo "✔ Wrote: $OUT_FILE"
echo

echo "🧭 Mode → Model mapping (as configured by this installer):"
printf "  %-8s %s\n" "c-ask"   "$ASK_MODEL"
printf "  %-8s %s\n" "c-plan"  "$PLAN_MODEL"
printf "  %-8s %s\n" "c-safe"  "$AGENT_MODEL"
printf "  %-8s %s\n" "c-agent" "$AGENT_MODEL"
echo

cat <<'SNIPPET'
Next step (manual, safer): add ONE line to your ~/.zshrc

# ---- Copilot Modes ---------------------------------
source "$HOME/.config/copilot-modes/copilot-modes.zsh"
# ----------------------------------------------------

Then reload your shell:

  source ~/.zshrc

Try it:

  c-modes

Thin wrapper (you provide -p):
  c-ask   -p "Summarize stdin" < report.txt
  c-agent -p "Run tests" --allow-tool 'shell(git:*)' --deny-tool 'shell(git push)'

Convenience (adds -p automatically):
  cat report.txt | cp-ask "Summarize stdin in 5 bullets"
  cp-plan "Outline a plan to refactor this module"

To verify your actually-available models (dynamic, policy-dependent):
  copilot
  /models
SNIPPET

echo
echo "⚠️  Important reminder:"
echo "If you re-run this script or edit copilot-modes.zsh, you MUST re-source:"
echo "  source ~/.zshrc"
echo "or start a new terminal session."
echo
echo "Done."
3/ Make it executable

> chmod +x copilot-modes-installer.sh
4/ Run it with: 

> ./copilot-modes-installer.sh
5/ Follow the instructions that get output, and put the code inside your ~/.zshrc at the bottom as noted:

# ---- Copilot Modes ---------------------------------
source "$HOME/.config/copilot-modes/copilot-modes.zsh"
# ----------------------------------------------------
5/ Reload your ~/.zshrc with the command: 

> source ~/.zshrc
You now have access to something similar to Simon Willison’s popular llmCLI tool but it’s making use of the powerful harness that’s been designed within copilot to do all kinds of cool things. We’ll show you what that means in a moment.

From now, you should be able to do the following with a few new commands at your disposal that wrap four modes around the basic copilot incantation in the form of:

-ask: it won’t be able to do anything that impacts the folder you’re sitting in
-edit: it can edit the files inside the folder you’re sitting in
-safe: it can do “agentic” kind of things but it’s in safety mode
-agent: it’s been given all the keys to your kingdom to cook away

One mode is similar to copilot -p "prompt" and looks like:

> c-ask   -p "Summarize stdin" < report.txt
> c-agent -p "Run tests" --allow-tool 'shell(git:*)' --deny-tool 'shell(git push)'
Note that the various parameters that normally get sent to copilot pass-through in this mode. And there’s a convenience mode where the -p isn’t needed if you want to do something more like you’re prompting copilot :

> cat report.txt | cp-ask "Summarize stdin in 5 bullets"
> cp-plan "Outline a plan to refactor this module"
If you don’t have a file report.txt in your folder just say: 

> cp-edit "make a file 'report.txt' with a concise fictional business report on ai"
And it will magically create a file for you to play with!

Keep in mind we’re just using UNIX-style thinking here to connect prompts and be sure to:

Use < file to define where input comes from
Use | to define what happens next

And each c-* command is just copilot with a different contract.

Note that you can change the different model you might want to use to do the various modes. That’s the trick you’ll want to adjust later.
